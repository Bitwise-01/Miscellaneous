What is Git?

Git is just one of many version control software out there. Git is a distributed version control system, meaning multiple people can be working on the same code locally without affecting the main project. Git is also designed in a way where someone can work on the code offline. When someone is done editing the code, they can use git to upload the code to a website called Github. These are the reasons why Git is a well known version control software because it makes version control easier for software developers.

A few Git functions

  • Pull
  • Clone
  • Fetch

Fetch

^

What is it?

Fetch downloads new data from a remote projects - example Github, but it doesn't add any of fetched data into your local project, the one that you have cloned. Fetch is great for getting a view on all the updates that took place in a remote project. Fetch is written in a way where it will never corrupt existing file on your local.

Pull

^

What is it?

Unlike fetch, pull is used to update your current local project with the latest changes from Github. This means that pull not only downloads the latest data from Github; it also directly adds it into your current working copy files.

Clone

^

What is it?

Clone is used to point to an existing project and make a local copy of that project. The original project can be located on the local filesystem or on Github. The clone command copies an existing project and reneder it and then update it to the original project.
